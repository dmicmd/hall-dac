# ТЗ для агента: прошивка датчика усилия на ATmega48P (Hall + LUT + PWM)

## 1. Общая задача

Разработать прошивку для ATmega48P, реализующую "умный" датчик усилия:

* Измерение сигнала с датчика Холла (аналоговый вход).
* Режим калибровки при включении питания.
* Формирование LUT (10 точек).
* Линейная интерполяция между опорными точками.
* Выходной сигнал — ШИМ 32 кГц на OC1A с последующей RC-фильтрацией.
* Индикация состояния через светодиод.

Прошивка должна быть написана на C (avr-gcc), без Arduino, без HAL.

## 2. Микроконтроллер

* MCU: ATmega48P
* Тактовая частота: 8 МГц (внутренний RC)
* Инициализация clock prescaler при необходимости

## 3. Распиновка

| Порт       | Назначение                        |
| ---------- | --------------------------------- |
| PC0 (ADC0) | Измерительный вход (датчик Холла) |
| PC3 (ADC3) | Калибровочный вход                |
| PB1 (OC1A) | Выход ШИМ 32 кГц                  |
| PD5 (OC0B) | Светодиод (активный HIGH)         |

## 4. АЦП

* Разрядность: 10 бит
* Опорное напряжение: AVCC
* Выравнивание: правое
* Предделитель: под частоту АЦП ~125 кГц
* Режим: одиночные измерения

Функция для чтения АЦП:

```c
uint16_t adc_read(uint8_t channel);
```

## 5. Условие входа в режим калибровки

При старте системы:

1. Считать ADC3 (PC3).
2. Если значение находится в диапазоне 0x03A0 .. 0x03B4 → перейти в режим калибровки.

## 6. Логика калибровки

### 6.1 Общая логика

* Постоянно считывать ADC3.
* При попадании значения в заданное "окно" захватывать текущее значение ADC0.
* Сохранять в LUT.
* После захвата точки ждать выхода из окна → переход к следующему.
* Всего 10 точек.

### 6.2 Окна захвата

| Индекс                                                   | Диапазон (hex)   |
| -------------------------------------------------------- | ---------------- |
| 0                                                        | 0x004C .. 0x0060 |
| 1                                                        | 0x00A1 .. 0x00B5 |
| 2                                                        | 0x00F7 .. 0x010A |
| 3                                                        | 0x014C .. 0x015F |
| 4                                                        | 0x01A1 .. 0x01B4 |
| 5                                                        | 0x01F6 .. 0x020A |
| 6                                                        | 0x024C .. 0x025F |
| 7                                                        | 0x02A1 .. 0x02B4 |
| 8                                                        | 0x02F6 .. 0x0309 |
| 9                                                        | 0x034B .. 0x035F |
| (Последнее окно 0x03A0..0x03B4 только для входа в режим) |                  |

### 6.3 Защита от дребезга

* Значение должно находиться в окне минимум N последовательных измерений (8–16)
* Использовать фильтр (скользящее среднее 4–8 выборок)

### 6.4 Сохранение LUT

* После калибровки сохранять массив из 10 uint16_t в EEPROM
* Контрольная сигнатура для проверки валидности

```c
typedef struct {
    uint16_t signature;
    uint16_t lut[10];
} calib_data_t;
```

signature = 0xBEEF

## 7. Обычный режим работы

### 7.1 Загрузка калибровки

* Считать EEPROM
* Проверить signature
* Некорректно → использовать дефолтную линейную шкалу

### 7.2 Измерение

* Считать ADC0
* Интерполировать по LUT
* Формировать нормированный выход 0..1023

## 8. Интерполяция

* Линейная между соседними точками LUT:

```c
y = y0 + (x - x0) * (y1 - y0) / (x1 - x0)
```

* LUT[0] → 0, LUT[9] → 1023

## 9. Формирование ШИМ

* Timer1, Fast PWM 10-bit
* Частота ~32 кГц
* Выход OC1A (PB1)
* Обновление OCR1A для изменения ШИМ

## 10. Светодиод (PD5)

* Медленное мигание — нет калибровки
* Быстрое мигание — режим калибровки
* Постоянно горит — нормальная работа
* Использовать Timer0 или программный таймер

## 11. Защита от ошибок

* Игнорировать захват точек не по порядку
* Таймаут калибровки (например 30 секунд)
* Защита от деления на 0 при интерполяции

## 12. Требования к коду

* Чистый C (C99)
* Без Arduino
* Разделение на файлы: main.c, adc.c/.h, pwm.c/.h, calib.c/.h, eeprom_storage.c/.h
* Без динамической памяти
* Без float (целочисленная арифметика)

## 13. Дополнительно

* Фильтрация измерительного канала (IIR или среднее)
* Стабильная работа при дрожании входа
* Код компилируем avr-gcc без предупреждений
* Сборка в gitlab ci/cd
